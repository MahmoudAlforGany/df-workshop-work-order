How the tool works
Important: This tool is operationally controlled by Item Master. Spare parts descriptions, cost, price, category, brand, and UOM are automatically controlled by the master data source — they cannot be typed manually on a work order.
Item List is the single source of truth.
  • On the Work Order form, you only type Item No and Qty. Everything else auto-fills from Item Master.
  • If an Item No is unknown the line is highlighted red and the work order cannot be saved.
  • If an item is Inactive, the line is blocked unless Admin is enabled (top right — password required).
  • Item Master itself can only be edited by an Admin. Operators can read but not modify master data.
Master Data Source
  • The tool runs in Local Mode by default — items live in the browser.
  • An Admin can import the company-wide master via Settings → Master Data Source → Import Master CSV / JSON. The mode switches to Shared Master Loaded with a timestamp.
  • Future integrations: SharePoint, OneDrive, NAV / Business Central, company shared server. See the Shared Master Data Connector block in the source for connector stubs.
Admin Mode
  • Click the red Admin: OFF button in the header and enter the admin password.
  • Admin can: edit Item Master, edit Settings/Config, delete work orders, wipe data, override approval workflow, save lines with inactive items.
  • Normal users can create / update / search work orders without admin rights.
Create a Work Order
  • Click New, fill the header (WO No, Date, Branch, Equipment, Status, Department, Priority, Assessment).
  • Click + Add Line for each spare part. Type the Item No, then Qty.
  • Totals and Margin % calculate automatically.
  • Click Save. Every change is written to the logs.
Item Master Import (CSV)
  • Open Item Master → Import CSV or Settings → Master Data Source → Import Master CSV.
  • Expected columns (case- and spacing-tolerant): itemNo, itemDescription, category, brand, uom, unitCost, unitPrice, vendor, status, notes.
  • Existing Item Nos are updated; new ones are inserted. Duplicates within the file are skipped.
Logs / Audit Trail
  • Every create / update / delete / approval / status change is recorded.
  • Spare-part line changes are stored separately with Line Key = WO#|Line#.
  • Both logs can be exported as CSV for Power BI / SharePoint audits.
Export & Migration
  • Settings → Export All Data (JSON) for full backup.
  • CSV exports for Items, Work Orders, Spare Parts Lines, and Logs feed Excel / SharePoint / Power BI directly.
  • All data lives in browser localStorage as an offline fallback. Migrate to ERP/NAV by reading the same JSON structure.
Validation Rules
  • WO No, Date, Branch, Equipment Item No, Status, Department, Priority, Assessment are required.
  • Spare part lines require a valid, active Item No and Qty > 0.
  • Approval requires Status = Completed or Cancelled (Admin can override).
  • WO cannot move to Completed if any spare part line has unit cost = 0 (warns).
